# Руководство по использованию сайта Nord City

## 1. Назначение системы

Сайт Nord City — это административная панель для управления данными бизнес-центра и связанными сценариями Telegram-бота. Через сайт можно:

- работать с пользователями;
- просматривать и редактировать заявки на обслуживание;
- обрабатывать гостевую парковку;
- вести объекты и помещения;
- просматривать отзывы;
- настраивать опросы;
- отправлять массовые уведомления пользователям через бота;
- работать с файлами;
- управлять локализацией и настройками бота;
- просматривать журнал аудита изменений.

Сайт рассчитан в первую очередь на администраторов и super admin.

---

## 2. Вход в систему

### 2.1. Авторизация

Вход выполняется по Telegram:

1. Откройте страницу входа.
2. Введите ваш `Telegram ID` или `@username`.
3. Нажмите `Получить код`.
4. Бот отправит вам 6-значный код в Telegram.
5. Введите код в форме подтверждения.

Особенности:

- код действует ограниченное время;
- если код не пришёл, его можно отправить повторно;
- доступ к панели предоставляется только тем пользователям, которым разрешён вход.

## 3. Общая структура сайта

После входа слева отображается боковое меню с основными разделами:

- `Панель управления`
- `Пользователи`
- `Заявки на обслуживание`
- `Гостевая парковка`
- `Отзывы`
- `Опросы`
- `Оповещения`
- `Бизнес-центры`
- `Журнал аудита`
- `Хранилище файлов`
- `Локализация бота`

В верхней части каждой страницы обычно отображаются:

- краткое описание;
- кнопки действия, если они доступны по роли.

---

## 4. Общие принципы работы с интерфейсом

### 4.1. Таблицы

Во многих разделах используется единый формат таблиц. Таблицы поддерживают:

- поиск;
- фильтрацию;
- сортировку;
- пагинацию;
- переключение видимости колонок;
- массовое выделение строк;
- экспорт данных;
- контекстные действия по правому клику или через меню строки.

Типовые действия в таблицах:

- открыть карточку записи нажатием по строке;
- перейти в режим редактирования;
- удалить запись;
- скопировать данные записи;
- отфильтровать список по нужным полям.

### 4.2. Карточки и детальные страницы

У большинства сущностей есть отдельная страница просмотра:

- пользователь;
- заявка на обслуживание;
- заявка на парковку;
- объект;
- помещение;
- отзыв.

Обычно на детальной странице можно:

- посмотреть все ключевые поля;
- перейти к связанным сущностям;
- выполнить редактирование, если хватает прав.

### 4.3. Формы

Формы создания и редактирования используются для:

- пользователей;
- заявок;
- парковки;
- объектов;
- помещений;
- отзывов.

Стандартный порядок работы:

1. Открыть форму создания или редактирования.
2. Заполнить обязательные поля.
3. При необходимости прикрепить файлы или изображения.
4. Нажать `Сохранить`.

### 4.4. Уведомления

На сайте используются всплывающие уведомления. Через них система сообщает:

- об успешном сохранении;
- об ошибках загрузки;
- об ошибках валидации;
- о результатах удаления, экспорта и отправки уведомлений.

## 5. Панель управления

Раздел `Панель управления` — это стартовый экран после входа.

На странице отображаются:

- сводка по пользователям;
- количество заявок;
- количество заявок в работе;
- количество завершённых заявок;
- количество отзывов;
- число объектов;
- число помещений;
- число свободных помещений;
- дневная сводка;
- графическая аналитика.

Для обновления данных используется кнопка `Обновить данные`.

Когда использовать:

- для общего контроля состояния системы;
- для быстрой оценки текущей нагрузки;
- для перехода в проблемные зоны, если показатели изменились.

---

## 6. Пользователи

Раздел `Пользователи` нужен для ведения списка пользователей системы и Telegram-бота.

В таблице обычно доступны:

- ID;
- ФИО и username;
- контакты;
- роль;
- объект;
- юридическое лицо;
- дата создания.

### Что можно делать

- открыть карточку пользователя;
- создать нового пользователя;
- отредактировать пользователя;
- удалить пользователя;
- экспортировать список;
- отфильтровать пользователей по объекту и другим полям.

### Особенности

- объект пользователя связан с бизнес-центром;
- юридическое лицо хранится отдельно и используется для идентификации;
- роль влияет на права доступа и поведение в боте.

### Ссылки ролей

На странице `Пользователи → Ссылки ролей` super admin может:

- получить deep-link ссылки для назначения ролей в боте;
- скопировать ссылку;
- открыть ссылку напрямую.

Этот раздел нужен для первичной выдачи доступа пользователям через Telegram.

---

## 7. Заявки на обслуживание

Раздел `Заявки на обслуживание` используется для просмотра и сопровождения заявок пользователей.

В списке заявок обычно отображаются:

- ID;
- описание заявки;
- местоположение;
- пользователь;
- объект;
- статус;
- дата создания.

### Что можно делать

- открыть карточку заявки;
- создать заявку вручную;
- редактировать заявку;
- удалить заявку;
- экспортировать список;
- фильтровать заявки по пользователю, объекту, статусу и датам.

### Статусы

В системе используются основные статусы:

- `NEW`
- `ASSIGNED`
- `IN_PROGRESS`
- `ACCEPTED`
- `COMPLETED`

Названия и цветовые метки отображаются в русифицированном виде в интерфейсе.

### Важный сценарий

Часть жизненного цикла заявок может происходить через Telegram-бота и админ-чат:

- бот отправляет уведомление о новой заявке;
- администратор отвечает на сообщение в чате;
- статус заявки обновляется;
- пользователь получает обратную связь через бота.

---

## 8. Гостевая парковка

Раздел `Гостевая парковка` используется для ведения заявок на парковочные места для гостей арендаторов.

В таблице доступны:

- ID;
- дата и время заезда;
- госномер;
- марка и цвет автомобиля;
- арендатор.

### Что можно делать

- открыть заявку;
- создать заявку вручную;
- отредактировать заявку;
- удалить заявку;
- фильтровать и экспортировать список.

### Особенность интерфейса

Если до времени заезда остаётся немного времени, заявка может подсвечиваться в списке для более быстрого внимания со стороны администратора.

### Настройки гостевой парковки

В разделе `Гостевая парковка → Настройки` можно:

- загрузить до двух изображений схемы проезда;
- изменить изображения маршрута;
- сохранить настройки.

Эти изображения затем могут использоваться в пользовательском сценарии через бота.

---

## 9. Отзывы

Раздел `Отзывы` предназначен для просмотра и обработки отзывов от пользователей.

Через интерфейс можно:

- открыть конкретный отзыв;
- редактировать отзыв;
- удалять отзыв;
- фильтровать отзывы;
- экспортировать список.

Отзывы помогают отслеживать качество работы и обратную связь от арендаторов.

---

## 10. Опросы

Раздел `Опросы` используется для просмотра результатов и управления опросной логикой.

Через страницу опросов можно:

- просматривать собранные ответы;
- фильтровать и анализировать список;
- при наличии прав управлять связанными данными.

### Настройки опросов

В разделе `Опросы → Настройки` можно:

- задать текст сообщения, которое отправляется пользователю;
- указать ссылку на Google Форму;
- проверить, что ссылка в тексте только одна;
- сохранить обновлённое сообщение.

Ограничение:

- сообщение должно содержать ровно одну ссылку;
- ссылка должна вести на Google Forms (`forms.gle` или `docs.google.com/forms/...`).

---

## 11. Оповещения

Раздел `Оповещения` используется для массовой отправки сообщений пользователям через Telegram-бота.

### Что можно настроить

- получателей по ролям;
- получателей по конкретным пользователям;
- заголовок сообщения;
- текст сообщения;
- вложения.

### Поддерживаемые вложения

- изображения;
- документы;
- PDF;
- DOCX;
- TXT;
- MD;
- Excel-файлы.

### Порядок отправки

1. Нажмите `Настройка отправки`.
2. Выберите роли и/или конкретных пользователей.
3. Укажите заголовок.
4. Введите текст.
5. При необходимости приложите файлы.
6. Нажмите `Отправить уведомление`.

После отправки система покажет результат:

- успешно доставлено;
- частично доставлено;
- не удалось доставить.

---

## 12. Бизнес-центры

Раздел `Бизнес-центры` используется для управления объектами недвижимости и вложенными помещениями.

В этой части системы обычно можно:

- просматривать список объектов;
- открывать карточку объекта;
- редактировать объект;
- создавать новый объект;
- просматривать помещения внутри объекта;
- переходить в карточку конкретного помещения;
- редактировать помещение;
- управлять фотографиями и описанием.

### Что хранится в объектах

- название;
- адрес;
- описание;
- статус;
- фотографии.

### Что хранится в помещениях

- привязка к объекту;
- этаж;
- площадь;
- описание;
- статус;
- изображения.

Эти данные используются как в админке, так и в сценариях Telegram-бота.

---

## 13. Журнал аудита

Раздел `Журнал аудита` показывает историю изменений в системе.

По каждой записи можно увидеть:

- тип сущности;
- ID сущности;
- действие;
- кто выполнил изменение;
- источник;
- дату и время.

### Для чего нужен аудит

- отслеживание действий администраторов;
- разбор спорных ситуаций;
- контроль изменений заявок, пользователей, файлов и настроек;
- анализ истории редактирования.

Обычно при нажатии на строку открывается подробная информация по событию.

---

## 14. Хранилище файлов

Раздел `Хранилище файлов` — это системный реестр всех файлов, загруженных через storage layer.

В таблице можно увидеть:

- ID файла;
- исходное имя;
- storage path;
- тип;
- категорию;
- связанную сущность;
- MIME-тип;
- дату создания.

### Что можно делать

- открыть файл по публичной ссылке;
- посмотреть, к какой сущности он привязан;
- удалить файл;
- отфильтровать и экспортировать список.

### Что важно помнить

- удаление файла удаляет его из storage;
- файл отвязывается от сущности;
- действие необратимо.

---

## 15. Локализация бота

Раздел `Локализация бота` нужен для редактирования текстов, которые используются Telegram-ботом.

Через страницу можно:

- просматривать словарь локализации;
- искать ключи;
- открывать редактирование конкретного текста;
- сравнивать текущее и изменённое значение;
- сохранять изменения.

### Особенности

- тексты могут содержать HTML-разметку;
- тексты могут содержать placeholders вида `{?}`;
- количество placeholders нельзя менять без понимания, как они используются в коде;
- перед сохранением стоит проверять, где этот ключ используется.

### Когда использовать

- изменение текста кнопок;
- изменение шаблонов уведомлений;
- изменение подсказок в боте;
- корректировка сообщений об ошибках или успешных действиях.

---

## 16. Настройки бота

Раздел `Настройки бота` управляет доступностью отдельных функций Telegram-бота.

В интерфейсе отображаются карточки функций с переключателями. Для каждой функции можно включить или отключить:

- `Профиль`
- `Обслуживание`
- `Опрос`
- `Обратная связь`
- `Гостевая парковка`
- `Свободные площади`

### Что происходит при отключении функции

- кнопка исчезает из главного меню бота;
- соответствующий раздел становится недоступен пользователю;
- изменения применяются сразу после сохранения.

### Как работать

1. Откройте страницу `Настройки бота`.
2. Включите или выключите нужные функции.
3. Нажмите `Сохранить`.

Если передумали, можно отменить изменения до сохранения.

---

## 17. Роли и доступ

Точные права зависят от вашей роли. В текущем интерфейсе встречаются как минимум два уровня:

- `Администратор`
- `Super Admin`

### Обычно администратору доступны

- просмотр и редактирование основных сущностей;
- работа с заявками;
- работа с парковкой;
- отправка уведомлений;
- редактирование локализации;
- изменение настроек бота.

### Обычно только super admin доступны

- создание пользователей в расширенном виде;
- управление ссылками ролей;
- отдельные расширенные административные действия.

Если раздел не открывается или на странице написано `Доступ ограничен`, значит текущая роль не позволяет это действие.

---

## 18. Рекомендуемый порядок повседневной работы

Ниже пример базового рабочего сценария администратора:

1. Открыть `Панель управления` и посмотреть сводку за день.
2. Проверить `Заявки на обслуживание`.
3. Проверить `Гостевую парковку`.
4. При необходимости открыть `Отзывы`.
5. Если нужно уведомить пользователей, перейти в `Оповещения`.
6. Если нужно поменять тексты бота, открыть `Локализация бота`.
7. Если нужно временно отключить функцию бота, зайти в `Настройки бота`.

## 19. Частые задачи

1. Перейти в `Пользователи`.
2. Нажать `Добавить пользователя`.
3. Заполнить данные.

### Найти проблемную заявку

1. Перейти в `Локализация бота`.

3. Отредактировать значение.
4. Сохранить.

### Отключить раздел в боте

1. Перейти в `Настройки бота`.
2. Выключить соответствующий переключатель.
3. Сохранить.

### Разослать сообщение арендаторам

1. Перейти в `Оповещения`.
2. Настроить получателей.
3. Заполнить заголовок и текст.
4. При необходимости прикрепить файлы.
5. Отправить уведомление.

## 20. Что делать при ошибках

Если раздел не загружается или действие не выполняется:

- обновите страницу;
- проверьте, хватает ли прав вашей роли;
- проверьте обязательные поля в форме;
- убедитесь, что прикладываемый файл подходит по типу;
- проверьте, что ссылка в настройках опросов ведёт именно на Google Form;
- если ошибка связана с ботом, проверьте соответствующие тексты в локализации и настройки бота.

Если проблема остаётся:

- зафиксируйте, на каком разделе и действии она возникает;
- сохраните текст ошибки;
- при необходимости сверяйтесь с `Журналом аудита`.
